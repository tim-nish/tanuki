Canonical discussion: https://github.com/tim-nish/tanuki/issues/13

As a Tanuki operator running an unattended loop on a hostless target,
I want every driven scenario to execute the disposable plugin clone (never my
real working checkout), so that cumulative fixes are actually dogfooded and no
scenario reaches outside its sandbox.

## Acceptance criteria

- **Given** a hostless scenario, **when** the driver launches it, **then** its
  working directory is the plugin clone (or the clone's absolute path is
  injected into the charter) so a relative `tools/…` invocation resolves inside
  the sandbox, never to the operator's real checkout.
- **Given** a completed scenario, **when** the post-run isolation check runs,
  **then** it asserts no binary outside the workspace/clone was executed —
  failing the run if one was (assert *what was run*, not only *what was left
  dirty*).
- **Given** a loop that landed a fix on the integration branch, **when** the
  next iteration drives, **then** it exercises that fixed code (verify-by-absence
  is valid again).
- **Regression test** covers a hostless scenario and asserts the real checkout
  path is never invoked.
