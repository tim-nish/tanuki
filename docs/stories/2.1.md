Umbrella: https://github.com/tim-nish/tanuki/issues/15 (spec:
`specs/spec-host-snapshot/SPEC.md`, ratified 2026-07-15).

As a Tanuki operator running an unattended hosted loop,
I want the run to observe an immutable host fixture pinned at init,
so that every iteration dogfoods the identical host and my own concurrent
work in the real host repo is invisible to the run by construction.

## Acceptance criteria

- **Given** a hosted target, **when** `tanuki-loop init` runs, **then** it
  clones the live host once into `<run-dir>/host-base`
  (`git clone --quiet --no-hardlinks`), records `host_base_sha` and
  `host_fixture_path` in `state.json` (additive keys), and fails closed if
  the clone fails. Hostless targets are untouched.
- **Given** an initialized hosted run, **when** any iteration drives,
  **then** the loop passes `--host <run-dir>/host-base` to `tanuki-drive` —
  never the scenarios file's live path.
- **Given** a hosted target, **when** `tanuki-loop doctor` runs, **then** a
  fifth check verifies the host path resolves and is cloneable (throwaway
  clone, removed afterward — the baseline-test worktree pattern); fatal for
  hosted targets, skipped for hostless.
- **Given** init, **when** host freshness is assessed, **then** it is
  read-only (`git ls-remote` compare) and informational only — init never
  fetches in the live host and never fails closed on host staleness.
- **Given** `tanuki-loop finish`, **when** the run closes, **then** the one
  permitted post-init read of the live host (`git rev-parse HEAD`) produces
  an informational drift note ("live host moved during the run: <sha> →
  <sha>") — never a breaker, never a reconciliation.
- **Given** `commands/tanuki-loop.md`, **then** it carries the out-of-scope
  rule: after init the skill never inspects or modifies any path outside the
  run dir and the loop worktree — no real-host reads, diffs, writes, or
  "restores"; remediation of real trees is not a loop capability.
- **Regression test** covers: fixture created at init with recorded SHA;
  a commit to the live host after init does not change what a later drive
  clones; finish emits the drift note.
