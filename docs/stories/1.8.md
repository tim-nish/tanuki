Issue: https://github.com/tim-nish/tanuki/issues/74 (canonical discussion
record — on conflict, the issue wins).

Origin: the `/tanuki` grammar has no stated rule for what is a bare word and
what is a flag, and is already mixed — `init` is a word, every other mode is
a flag, nothing says why. With no rule, each new mode is answered by local
precedent, which is how two changes drafted in one sitting (2026-07-17) came
out opposite: `--decide` following `--brief`, a proposed `view` following
`init`. Spec decision executed 2026-07-17: ratify the rule once in
`specs/spec-short-command-surface/SPEC.md` D6 and apply it to every command
in a single implementation, rather than fixing instances one at a time and
churning entry points through repeated renames.

The rule (D6), stating what the repo already practices — every non-driving
mode's doc says "No driving", and the one word in the grammar is the one mode
that never drives:

> A bare word selects a mode that does not drive. Flags modify a drive. The
> bare default is driving.

As an operator, I want one rule to tell me whether a mode is a word or a
flag, so that I can predict the surface instead of memorizing it — and so
the next mode added doesn't reopen the question.

## Acceptance criteria

- **Given** `commands/tanuki.md`'s argument grammar, **then** every
  non-driving mode is a bare word: `init`, `decide`, `status`, `history`,
  `view`, `mine <run>`, `ingest "<text>"`. Driving stays the bare default
  (`/tanuki <t>`, `<scenario-ids>`, `"<free text>"`).
- **Given** any documented flag spelling (`--brief`, `--status`,
  `--history`, `--mine-only`, `--ingest`), **when** an operator uses it,
  **then** it still works as an alias and is documented as such — nothing in
  an operator's fingers breaks.
- **Given** `--brief`, **then** it aliases `decide` specifically (reprinting
  the latest brief, then continuing into the pass — the story 1.7 contract).
- **Given** the remaining flags, **then** only true modifiers survive as
  flags (`--follow`, `--json`, scenario selection); no flag selects a
  non-driving mode except as a retained alias.
- **Given** `docs/tanuki-spec.md` §Command, **then** it states the rule and
  lists the word grammar with the aliases named.
- **Given** a reader of `specs/spec-short-command-surface/SPEC.md`, **then**
  D6 is the single place the rule is stated; no other doc restates it.
- **Given** `view` is not yet implemented (`specs/spec-tanuki-view/SPEC.md`
  is PROPOSED), **then** this story ratifies its *shape* only — resolving
  that spec's OPEN-3 — and does not implement the view surface.
- Test coverage in `tools/tests/` where the grammar is mechanically checkable;
  otherwise the acceptance is the documented grammar itself.
