Umbrella: https://github.com/tim-nish/tanuki/issues/15 (spec:
`specs/spec-host-snapshot/SPEC.md`, ratified 2026-07-15).

As a Tanuki operator,
I want the drive to fail any scenario whose `host_setup` or commands reached
the clone-source path (the live host or fixture it cloned from),
so that an absolute path in a scenarios file can never read or write a tree
outside the sandbox — enforced, not assumed.

## Acceptance criteria

- **Given** a hosted scenario whose `host_setup` contains an absolute path to
  the clone source, **when** the drive runs it, **then** the scenario fails
  with a distinct non-ok status (clone-source access), the offending command
  is recorded as an event with evidence, and the manifest carries a boolean
  flag — the same assert-what-was-touched posture as the hostless
  execution-escape check (#13).
- **Given** a scenario whose transcript shows a command referencing the
  clone-source path, **when** the post-run isolation check runs, **then** the
  same failure applies (scenario commands are checked, not only
  `host_setup`).
- **Given** a scenario using only sandbox-relative paths, **when** it runs,
  **then** no flag is raised and the status stays `ok` (no false positives
  from the clone's own path, which lives under the workspace).
- **Regression test** extends `tools/tests/test-drive-isolation` with the
  hosted clone-source escape case and the clean hosted case.
