Issue: https://github.com/tim-nish/tanuki/issues/73 (canonical discussion
record — on conflict, the issue wins).

Origin: `specs/spec-tanuki-solve/SPEC.md` D2 names evidence-pointer overlap
among `consolidate`'s clustering signals, but the implemented
`tanuki-ledger consolidate` clusters on title-token similarity, subject
tokens, and scenario overlap only. `notes[]` — which often carries the
proposal nuance that relates or distinguishes two findings — and run-level
evidence overlap are unused. Merge candidates whose titles diverge but whose
notes or evidence align are missed, and the command layer has to catch them
by hand. D2 permits the command layer to add groups the clustering missed,
so this is degraded recall, not a correctness bug.

As an operator running a decision pass, I want `consolidate` to cluster on
what findings *say* and *where they came from* — not just how their titles
are worded — so that conflicting proposals reach one approval screen instead
of surviving as separate issues.

## Acceptance criteria

- **Given** two findings whose titles share few tokens but whose `notes[]`
  describe the same defect, **when** I run `tanuki-ledger consolidate`,
  **then** they appear in one candidate group and the group's reason names
  the notes-text signal.
- **Given** two findings whose evidence pointers share one or more run ids,
  **when** I run `consolidate`, **then** the shared run id counts as a
  corroborating signal alongside scenario overlap, and the reason names it.
- **Given** a run id shared by many unrelated findings, **then** run overlap
  alone never forms a group — it corroborates, and the never-groups-alone
  rule from D2 still holds.
- **Given** the same ledger content twice, **when** I run `consolidate`
  twice, **then** the output is byte-identical: fixed tokenizer, fixed
  thresholds, signals named per group (D2 determinism — no LLM in the inner
  loop).
- **Given** any invocation, **then** `consolidate` still reads bounded views
  only and writes nothing (D2).
- Test coverage in `tools/tests/test-ledger-consolidate` exercising:
  notes-only match (grouped), run-overlap corroboration (grouped with
  scenario), run-overlap alone (not grouped), and determinism across two
  runs.
